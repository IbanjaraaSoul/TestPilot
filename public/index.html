TestPilot

Paste a story or MR → AI generates test cases → run them in the browser.

Generated test cases

For good coverage the AI should produce at least 3 cases: happy path, invalid/negative case, and a validation case (e.g. empty fields).

Tip: The test browser opens on the machine where the server is running. Start the server from a local terminal (e.g. Terminal.app) with npm start — not from a remote/SSH session — so a Chromium window can appear. Check other desktops or use Cmd+Tab if you don’t see it. While tests run, the panel below shows live suite pass/fail counts and the current step; full step logs appear in the summary after the run finishes.
